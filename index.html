Save to repo (requires GitHub PAT)
PAT needs Contents: write on this repo. Never stored - only used for this request.
1
Open GitHub Settings
Go to github.com → your avatar (top right) → Settings
2
Developer Settings
Scroll to the very bottom of the left sidebar → click Developer settings
3
Fine-grained personal access tokens
Click Personal access tokens → Fine-grained tokens → Generate new token
4
Configure the token
Token nameTobias dashboard
Expiration90 days (or custom)
Resource owneryour GitHub username
Repository accessOnly select repositories → pick tobias
5
Set the one permission needed
Under Repository permissions, find Contents and set it to Read and write. That's the only permission required - leave everything else as No access.
6
Generate and copy
Click Generate token. Copy the token immediately - GitHub only shows it once. It starts with github_pat_
7
Paste above and save
Paste the token in the field above and click 💾 Save Config. The token is used only for that one API call and is never stored anywhere.
!
When does the change take effect?
Immediately on save - the next agent run will read data/config.json from the repo and use the new models. The Analyst runs at 11:00 UTC, Scout at 14:00 UTC.
!
Also used for: GitHub Actions secrets (one-time setup)
Your main PAT (the one with Contents write used above for daily config changes) is different from the GH_PAT secret stored in GitHub Actions. That one is set once in Repo → Settings → Secrets and variables → Actions → New repository secret and is used by the agents to commit data/state.json and data/history.json back to the repo after each run.
